# 라이프존 연속 기록 전광판 에셋 계획

`docs/ai/NPC_ASSET_WORKFLOW.md`의 공간/소품 체크리스트를 전광판 오버레이에 적용한다.

1. 홈 위치: 좌측 상단의 넓은 투명 여백(헬스장 상단 파사드 바깥) 전체다. 라이프존 1672 기준으로는 화면 좌측 약 0~830px, 상단 약 0~320px의 삼각형 여백을 전광판 판면으로 채운다.
2. 겹침: 별도 흰 카드나 벽 패치를 만들지 않는다. 전광판은 베이스룸 PNG의 뒤 레이어에 두며, 알파가 있는 파사드가 전광판의 아래·오른쪽을 자연스럽게 잘라낸다. NPC·운동 캐릭터·식탁 캐릭터·러닝트랙과는 겹치지 않는다.
3. 홈/모달 분리: NPC 작업이 아니므로 모달용 자산은 만들지 않는다. 홈 배치용 전광판 1종만 만든다.
4. 전용 소품: 빈 LED 화면을 가진 RGBA 전광판 PNG를 검은 파사드 위에 직접 올린다. 동적 문구와 숫자·정보 버튼만 DOM으로 렌더한다.
5. 원근: 베이스룸의 좌우 26~30도 아이소메트릭 축을 따르며, CSS 배경·테두리 없이 전광판의 검은 후면·드롭섀도만 파사드 바깥 여백에 연결한다.
6. 크기: 모바일에서는 장면 폭의 약 76%, 넓은 화면에서는 최대 540px까지 키운다. LED 유효 면은 약 145px 이상, 레이블은 9px 이상·일수는 23px 이상을 기준으로 한다.
7. 이름표/전구: 해당 없음. 연속 기록 문구, 수치, 안내 버튼은 기존 DOM과 이벤트를 유지한다.
8. 정적 자산: 전광판 PNG를 `assets/home/life-zone/ui/`에 두고 `runtime-assets.js`에 등록한다. 기존 베이스룸 PNG는 변경하지 않는다.
9. 캐시: PNG, JS, CSS가 바뀌므로 `sw.js`의 `CACHE_VERSION`을 같은 변경에서 올린다.
10. 완료 증거: `http://localhost:3000`의 HTTP 200과 로그인된 홈 화면에서 카드 좌측 절반 배치, 각도, 둥근 모서리 잘림을 확인한다. 운영 배포는 사용자가 명시적으로 요청할 때만 수행한다.
