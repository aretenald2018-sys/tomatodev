# 라이프존 연속 기록 전광판 렌더링 계획

## 최종 구조

1. 전광판은 카드·뷰포트 좌표가 아니라 `lz-world`의 고정 `1672 × 1672` 월드 좌표에서 렌더링한다.
2. 생성한 투명 PNG 프레임 `streak-marquee-frame-v3.png`과 `STREAK`·3자리 연속 기록·토마토 픽셀은 한 장의 캔버스에 그린다. CSS는 캔버스 전체를 균일하게 축소할 뿐, 전광판 안의 요소를 따로 변형하지 않는다.
3. 프레임과 비트맵 정보의 좌표·각도·스케일은 캔버스 안에서 한 번만 적용한다. 별도 DOM 문구, 역배율, 컨테이너 쿼리 단위, 중첩 `transform`을 사용하지 않는다.
4. 화면 표기는 작은 폭에서도 판독 가능한 고정 비트맵 정보(`STREAK`, `000`~`999`, 토마토)로 제한한다. 긴 동기 문구는 시각 판면에 억지로 넣지 않고, 전광판 전체를 덮는 접근 가능한 상세 버튼의 레이블과 규칙 안내에서 제공한다.
5. 버튼은 투명한 `#tomato-rule-info-card` 핫스폿 하나만 유지한다. 기존 토마토 획득 규칙 이벤트는 그대로 연결되며 키보드 포커스도 보인다.

## 자산·캐시

- 프레임 PNG는 `assets/home/life-zone/ui/`에 저장하고 `runtime-assets.js`에 등록한다.
- PNG, JS, CSS가 바뀌면 같은 변경에서 `sw.js`의 `CACHE_VERSION`을 올린다.
- 베이스룸 아트는 수정하지 않는다. 전광판만 투명 캔버스로 레이어링한다.

## 완료 증거

- 실제 브라우저 `320px`, `456px`, `768px`, `1024px` 폭에서 캔버스가 준비된 뒤 캡처한다.
- 프레임과 숫자가 같은 전광판 면에 있고, 상단·좌우가 잘리지 않으며, 버튼이 규칙 안내를 여는지 확인한다.
- 운영 배포는 위 캡처를 확인한 뒤에만 수행한다.
