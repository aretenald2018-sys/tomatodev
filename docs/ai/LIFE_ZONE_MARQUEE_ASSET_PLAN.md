# 라이프존 연속 기록 전광판 에셋 계획

`docs/ai/NPC_ASSET_WORKFLOW.md`의 공간/소품 체크리스트를 전광판 오버레이에 적용한다.

1. 홈 위치: 전광판을 카드 좌표가 아니라 `lz-world`의 고정 `1672x1672` 좌표에 배치한다. 파사드 기준값은 `x=-17`, `y=-20`, `w=920`이며 좌측 상단의 삼각형 여백을 판면으로 채운다.
2. 겹침: 파사드 이미지와 DOM 문구를 하나의 `lz-marquee-composite` 안에 렌더해 동일한 위치·크기·변형을 공유한다. 합성 컨테이너를 베이스룸 PNG보다 아래 레이어에 두고, 베이스룸 알파가 전광판과 문구를 함께 자연스럽게 잘라낸다.
3. 홈/모달 분리: NPC 작업이 아니므로 모달용 자산은 만들지 않는다. 홈 배치용 전광판 1종만 만든다.
4. 전용 소품: 지지대 없는 RGBA 파사드 전광판 `streak-marquee-facade-v2.png`를 사용한다. 빈 LED 화면 위의 문구·숫자·정보 버튼은 파사드 원본의 정규화 좌표로 DOM 렌더링한다.
5. 원근: 베이스룸의 좌우 26~30도 아이소메트릭 축을 따른다. 합성 컨테이너 전체를 세로 `1.55`배로 투영하고, DOM 문구에는 정확한 역배율 `0.645161`과 `-20deg` 회전을 적용한다. 따라서 아트와 문구는 같은 판면을 따르면서 글자 자체는 찌그러지지 않는다.
6. 크기: 전광판 너비는 항상 원본 월드의 `920/1672` 비율로 계산한다. 문구 컨테이너는 별도의 월드 좌표를 갖지 않고 원본 이미지 안의 `left=13.3%`, `top=55.7%`, `width=69.6%`, `height=26%`를 유지한다. 카드 폭이 달라져도 이미지와 문구의 상대 위치는 변하지 않으며 세 요소는 카드 폭 단위(`cqw`)에 비례해 중앙 정렬한다.
7. 이름표/전구: 해당 없음. 연속 기록 문구, 수치, 안내 버튼은 기존 DOM과 이벤트를 유지한다.
8. 정적 자산: 새 파사드 PNG는 `assets/home/life-zone/ui/`에 저장하고 `runtime-assets.js`에 등록한다. 베이스룸 PNG는 수정하거나 교체하지 않는다.
9. 캐시: PNG, JS, CSS가 바뀌므로 `sw.js`의 `CACHE_VERSION`을 같은 변경에서 올린다.
10. 완료 증거: `320px`, `456px`, `768px`, `1024px` 실제 브라우저 뷰포트에서 전광판·문구·정보 버튼이 같은 판면 안에 유지되는지 캡처로 확인한다. 운영 배포는 사용자가 명시적으로 요청할 때만 수행한다.
