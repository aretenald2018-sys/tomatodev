# 라이프존 연속 기록 전광판 에셋 계획

`docs/ai/NPC_ASSET_WORKFLOW.md`의 공간/소품 체크리스트를 전광판 오버레이에 적용한다.

1. 홈 위치: 전광판을 카드 좌표가 아니라 `lz-world`의 고정 `1672x1672` 좌표에 배치한다. 파사드 기준값은 `x=-17`, `y=-20`, `w=920`이며 좌측 상단의 삼각형 여백을 판면으로 채운다.
2. 겹침: 전광판과 베이스룸을 같은 `lz-world` 안에서 함께 렌더한다. 전광판을 베이스룸 PNG보다 아래 레이어에 두고, 베이스룸 알파가 전광판의 아래·오른쪽을 자연스럽게 잘라낸다. NPC·운동 캐릭터·식탁 캐릭터·러닝트랙과는 겹치지 않는다.
3. 홈/모달 분리: NPC 작업이 아니므로 모달용 자산은 만들지 않는다. 홈 배치용 전광판 1종만 만든다.
4. 전용 소품: 지지대 없는 RGBA 파사드 전광판 `streak-marquee-facade-v2.png`를 사용한다. 빈 LED 화면 위의 문구·숫자·정보 버튼만 DOM으로 렌더한다.
5. 원근: 베이스룸의 좌우 26~30도 아이소메트릭 축을 따른다. 파사드 PNG는 모든 카드 폭에서 동일한 세로 `1.55`배로 투영해 900px 전후에서 전광판 두께가 갑자기 바뀌지 않게 한다. DOM 문구는 `-24deg`로 기울여 판면과 일치시킨다.
6. 크기: 전광판 너비는 항상 원본 월드의 `920/1672` 비율로 계산한다. 문구 컨테이너는 `x=105`, `y=205`, `w=560`, `h=120`을 유지해 320px 화면에서도 문구가 잘리지 않게 한다. 글자·토마토·정보 버튼은 최대값으로 막지 않고 카드 폭 단위(`cqw`)에 계속 비례시키며, 문구와 토마토를 하나의 중앙 정렬 그룹으로 배치한다. 문구만 별도로 위로 이동하지 않아 모든 화면에서 LED 판면 중앙에 남는다.
7. 이름표/전구: 해당 없음. 연속 기록 문구, 수치, 안내 버튼은 기존 DOM과 이벤트를 유지한다.
8. 정적 자산: 새 파사드 PNG는 `assets/home/life-zone/ui/`에 저장하고 `runtime-assets.js`에 등록한다. 베이스룸 PNG는 수정하거나 교체하지 않는다.
9. 캐시: PNG, JS, CSS가 바뀌므로 `sw.js`의 `CACHE_VERSION`을 같은 변경에서 올린다.
10. 완료 증거: `390px`와 `717px` 폭의 정적 합성에서 같은 월드 좌표로 배치되는지 확인하고, `http://localhost:3000`의 HTTP 200과 로그인된 홈 화면에서 전광판·문구·정보 버튼을 확인한다. 운영 배포는 사용자가 명시적으로 요청할 때만 수행한다.
