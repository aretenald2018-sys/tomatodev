# 운동 카드 접기/펼치기 UI 회귀 수정 계획

## 배경

- 사용자 제보: 운동 카드 접기/펼치기 중 버튼을 잘못 누르면 UI가 예전 버전처럼 운동 편집 화면으로 돌아간다.
- 첨부 화면은 운동 탭 편집 화면의 일반 세트 카드 UI이며, 새 캘린더/운동 상세 카드에서 편집 화면으로 튄 뒤 보이는 상태로 추정된다.

## /diagnose 재현 루프

1. 운동 탭의 월간 운동 홈에서 기록이 있는 날짜 상세를 연다.
2. 운동 상세 카드에서 `카드 접기`를 누른다.
3. 접힌 카드의 왼쪽 `운동 완료` 버튼을 누른다.
4. 현재 구현은 `window._wtCalEditSession()`을 호출해 운동 편집 화면으로 이동한다.
5. 기대 동작: 접힌 카드의 완료 상태 클릭은 편집 화면으로 이동하지 않고, 사용자가 명시적으로 `편집하기`를 누를 때만 편집 화면으로 이동한다.

## 원인 가설

1. 접힌 상세 카드의 `운동 완료` 버튼이 상태 표시처럼 보이지만 실제로는 `_wtCalEditSession()`에 바인딩되어 있어 오클릭 시 편집 화면으로 전환된다.
2. 편집 화면 진입 시 일반 운동 카드 렌더러가 노출되어 사용자가 "예전 버전 UI로 돌아감"으로 인지한다.
3. `카드 접기`/`세트 다시 보기` 자체의 collapse 상태 저장은 `_workoutDetailCollapsed` set과 `renderWorkoutCalendarHome()`로 동작하며, 직접적인 구 UI 렌더 원인은 아니다.
4. Max 편집 카드의 `_renderExerciseList()` 재렌더가 저장 콜백에서 반복되는 문제도 가능하지만, 첨부 화면과 "잘못 클릭" 증상은 캘린더 상세 카드 버튼 오동작과 더 잘 맞는다.

## 실행 Slice 1

- `render-calendar.js`의 접힌 운동 상세 카드에서 `운동 완료` 버튼을 편집 진입 버튼으로 사용하지 않는다.
- 접힌 상태의 기본 동작은 비활성 상태 표시로 두고, 펼치기는 기존 `세트 다시 보기` 버튼만 담당한다.
- 펼친 상태의 명시적 `편집하기`, 상단 `수정`, 플로팅 편집 버튼은 유지한다.

## 수정하지 않을 것

- 운동 편집 화면의 일반 세트 카드 디자인은 이번 범위에서 바꾸지 않는다.
- Max 모드 운동 편집 카드의 세트 완료/자동 접기 로직은 이번 범위에서 바꾸지 않는다.
- 캘린더 전체 레이아웃이나 월간 셀 동작은 바꾸지 않는다.

## 검증 기준

1. `node --check render-calendar.js`
2. `node --check sw.js`
3. `git diff --check`
4. 로컬 UI 플로우:
   - `npm.cmd run dev`
   - 운동 탭 월간 홈에서 기록 날짜 상세 열기
   - 카드 접기
   - 접힌 카드의 `운동 완료` 클릭 시 편집 화면으로 이동하지 않음
   - `세트 다시 보기` 클릭 시 같은 상세 카드가 펼쳐짐
   - 펼친 카드의 `편집하기` 클릭 시에만 운동 편집 화면으로 이동함
