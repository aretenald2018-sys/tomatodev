# 종목별 볼륨 그래프 기준 표시 및 일자 상세

## 그릴 결과

- 핵심 질문: 사용자가 말한 "기준"은 무엇을 화면에 더 드러내야 하는가?
- 답변/결정: 현재 그래프는 선택된 운동 종목의 날짜별 볼륨을 기준으로 한다. 볼륨은 완료된 본세트의 `kg × reps` 합계이며, ROM 보정값이 있으면 반영된다. 이 기준을 카드 안에 명시한다.
- 핵심 질문: 그래프 점 클릭 시 어떤 상세를 보여줘야 하는가?
- 답변/결정: 해당 날짜, 선택 종목명, 총 볼륨, 완료 본세트 수, 세트별 `kg × reps × RPE` 내역을 카드 내부 상세 패널로 보여준다. 별도 대형 모달 확대는 이번 슬라이스에 넣지 않는다.
- 남은 가정: 기존 `getCache()`의 `day.exercises[].sets` 데이터가 상세 내역의 소스다. Firestore 직접 호출 없이 기존 캐시만 사용한다.

## 실행 슬라이스 1

목표:
- `종목별 볼륨 추이` 카드에서 그래프의 기준을 즉시 알 수 있게 설명/선택 종목/계산식을 표시한다.
- 그래프의 점 또는 최근 기록 행을 클릭하면 해당일의 상세 운동내역을 같은 카드 안에서 확인할 수 있게 한다.

예상 변경 파일:
- `render-stats.js`
- `style.css`
- `sw.js`
- `docs/ai/NEXT_ACTION.md`

구현 범위:
- 기존 `Chart.js` 라인 차트에 point click handler를 추가한다.
- `getCache()`에서 선택 종목과 날짜의 운동 entry를 찾아 상세 패널을 렌더링한다.
- 최근 5회 기록 행도 같은 상세 패널을 열도록 버튼화한다.
- 기준 문구는 `kg × reps`, 완료 본세트, ROM 보정 여부를 짧게 표시한다.

비범위:
- 전체 화면 차트 확대 모달
- 운동 기록 데이터 모델 변경
- Firestore 직접 조회/저장
- 통계 탭 전체 재디자인

검증:
- `node.cmd --check render-stats.js`
- `npm.cmd run dev`
- 실제 URL에서 HTTP 200 확인
- 통계 탭 진입 후 `종목별 볼륨 추이` 카드에서 기준 문구가 보이는지 확인
- 그래프 점 클릭 또는 최근 기록 행 클릭 시 해당 날짜의 세트 상세가 표시되는지 확인
- `render-stats.js`와 `style.css`가 `STATIC_ASSETS`에 있으므로 `sw.js` `CACHE_VERSION`이 함께 변경됐는지 확인
