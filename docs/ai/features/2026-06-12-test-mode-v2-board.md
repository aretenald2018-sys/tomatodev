## rev 1.3 통합 메모 (실제 운동기록 연결)

- **종목 = 실제 레지스트리**: `benchmark.exerciseId`(getExList 원소 id) + `muscleId` 저장. `board-core.exerciseGroupId()`가 muscleId/subPattern/movementId로 그룹 판정. `buildRecentMap(getCache())`로 종목별 최근 본세트 무게 상속.
- **셀 탭 → 운동카드(`_openWorkoutCard`)**: 이번 주 칸만 카드(미래=계획 미리보기, 과거=기록 요약). 카드는 plan 처방으로 prefill(웬들러=메인 세트+BBB 세트). [운동 완료] 시 `WS.workout.exercises`에 entry 머지 + `saveWorkoutDay({silent})` → 실제 workouts 저장. 채워진 본세트 최고기록이 plan kg×reps 충족 시 `paintWeek`(자동 색칠), 미달이면 stair=조정 시트 / wendler=기록만.
- **RIR↔RPE**: 카드 RIR 입력 → 저장 시 `rpe = 10 - rir`. ROM → `romPct`. 운동 탭 세트 객체(`{kg,reps,rpe,romPct,setType,done}`)와 동일 shape.
- **18주 투영**: `projectFutureCells()` — 활성 이후 미래 사이클을 현재 대표 + 증량폭×offset로 계산(저장 안 함, 정산 때 실 사이클화). `_boardModel()`이 과거1+활성+투영2 밴드를 연속 그리드로 합치고 행 인덱스로 정렬, 오늘 주에 `.tm2-now-line`.

## 왜 v2인가 (요청 배경)

현재 테스트모드(Max V4, `workout/expert/max*.js`)는 기술부채가 심해 목업을 새로 만들어도 AI 구현이 stale 패턴(오늘 카드 중심 대시보드, 추천/조언 카드, 자동 트랙 교대)으로 회귀하는 현상이 반복된다. 그래서 v2는 **기존 코드를 고치지 않고 스크래치에서 새 모듈로 구축**한다. 이 문서가 의도의 단일 진실이며, 아래 "필수 구현 계약"이 회귀 방지 기준선이다.

- 목업: [docs/ai/mockups/2026-06-12-test-mode-v2-board.html](../mockups/2026-06-12-test-mode-v2-board.html) — 와이어프레임 + 서비스 시나리오 10개 씬 (씬 0~9)
- 원형: 사용자가 수년간 쓰던 엑셀 중장기 계획표 (열=종목, 행=주, 셀=중량/횟수, 달성하면 진한 녹색으로 직접 색칠)

## 보드의 이중 정체성 (rev 1.2 핵심)

보드는 두 가지를 동시에 해낸다. 이 둘을 분리해 별도 화면으로 만들면 회귀다.

1. **중장기 성장판**: 종목마다 6주 사이클로 무게가 계단식으로 오르는 계획표. 엑셀 그대로.
2. **당일 운동 배열판**: 부위 날에 보드를 열고 **오늘 행에서 할 종목을 골라 담는다** — 그게 오늘의 운동(배열·순서)이 된다. 같은 부위 날마다 똑같은 종목을 다 하는 게 아니다. 등록된 종목은 "그날 골라 쓰는 메뉴"이고, 고르지 않은 종목의 그 주 칸은 그냥 쉼이며 계획(칸)은 그대로 유지된다.

## 필수 구현 계약 (MUST — 어떤 세션도 이 13개를 우회하지 않는다)

1. **메인 화면은 보드(표)다.** 카드 대시보드·히어로·score-row·추천/조언 카드 형태로 만들지 않는다.
2. **행=주(월요일), 열=종목×트랙, 셀=중량×횟수.** 세트는 기본 4(종목별 변경 가능)이고 셀에 표기하지 않는다.
3. **볼륨/강도는 독립된 두 열이다.** 주차 자동 교대 로직을 만들지 않는다. 트랙 수는 종목별로 1~2개. UI 용어는 항상 "볼륨/강도".
4. **셀 상태 머신**: `빈칸(쉼) → 계획 → 이번주 → 달성(색칠) | 못 채움(조정)`. 달성 색칠은 자동이 아니라 **유저의 명시적 탭**이다 (수행 데이터 기반 제안은 가능하되 확정은 유저).
5. **같은 처방이 이어지면 셀을 병합한 스텝으로 표현**하고, 못 채웠을 때 1순위 조정은 "한 주 더 도전(스텝 1주 연장)"이다.
6. **6주 사이클 밴드가 보드에 항상 보인다.** 정산(성장/유지)도 보드 흐름 안(밴드 경계)에서 일어난다. 별도 대시보드로 빼지 않는다.
7. **정산 성장폭은 종목별 설정값(`incrementKg`)을 그대로 사용한다.** 코드에 +2.5/5 등 하드코딩 금지. 못 채운 주가 있던 종목은 "유지"가 기본 선택.
8. **지정 종목(주로 하체)은 웬들러 프로그램**: 기준 무게(TM) 기반 `weekMap[6]` 자동 처방(8/6/3 프리셋 + 5/3/1 + 커스텀) + BBB 보조. 웬들러 열의 셀은 모듈이 채우고(병합 없음, 주 단위), 유저는 색칠만 한다. **셀 시트는 한 세션의 순서 — 메인 8/6/3 세트 직후 바로 BBB — 를 그대로 보여준다.** 정산 = TM + 증량폭.
9. **자세 메모·여유 횟수(RIR)·헬스장별 기구는 종목 메타**로 존재하고, 셀 시트에서 항상 보인다. 색칠 기록에는 실제 횟수·RIR·메모를 남길 수 있다.
10. **줌아웃(6개월 간트 조망)을 제공한다.** "한 눈에 중장기 계획 가시화"가 이 제품의 존재 이유다.
11. **첫 설정(온보딩)이 있다.** 기존 테스트모드(v1) 벤치마크·운동 기록에서 **디폴트 종목을 자동 구성**해 보여주고, 시작 무게는 **기록이 있으면 상속, 없으면 직접 입력**받는다. 6주 증량 기본값은 **상체 대근육 +2.5kg / 하체 대근육 +10kg**으로 시드하되 종목별 수정 가능. 빈 보드에서 시작하게 하지 않는다.
12. **종목은 언제든 추가/삭제할 수 있다.** 추가는 기존 테스트모드 운동 라이브러리에서. 삭제는 보관(archived) 처리 — **색칠 기록은 보존**되고 다시 추가하면 이어서 시작한다.
13. **보드는 당일 운동 배열판이기도 하다.** 부위 날에 오늘 행에서 종목 칸을 골라 담으면 그것이 **오늘의 배열**(수행 순서 포함)이 된다. 부위 날마다 같은 구성을 강요하지 않는다. 고르지 않은 종목의 그 주 칸은 쉼이고 계획은 유지된다. 이 기능을 보드와 분리된 별도 화면으로 만들지 않는다.

## IA / 화면 (목업 씬과 1:1)

0. **씬 0 — 첫 설정**: ① 운동 메뉴 만들기(테스트모드에서 하던 종목 디폴트 구성 + 토글/추가 — "전부 매번 하는 게 아니라 그날 골라 쓰는 메뉴") ② 시작 무게 확인(기록 있으면 상속 표기, 없으면 직접 입력) + 6주 증량 기본값 안내(상체 +2.5 / 하체 +10) ③ 시작일 선택 → [6주 칸 채우기]. (계약 11)
1. **씬 1 — 보드 메인 + 오늘의 배열 (가슴 그룹)**: 그룹 칩, 날짜 레일 + 5열, 6주 사이클 밴드 2개, 정산 요약 스트립, 오늘 행 강조. **오늘 행의 칸을 탭해 담으면 ①②③ 순서 배지가 붙고 하단 "오늘의 배열" 바에 쌓인다** → [운동 시작]. 안 고른 종목은 그 주 쉼. (계약 13)
2. **씬 2 — 줌아웃 미니맵**: 핀치 아웃 → 5개 그룹 × 6개월 마이크로 셀 조망.
3. **씬 3 — 셀 시트 (달성 색칠)**: 처방 + 오늘의 기준(여유 횟수/자세 메모/기구 — 문장형) + 최근 기록 + 오늘 기록 입력 → [성공 — 칸 색칠하기].
4. **씬 4 — 못 채운 날 → 계획 조정**: 조정 옵션 3개(한 주 더 도전(추천)/무게 내리기/횟수 낮추기) + 적용 후 미리보기.
5. **씬 5 — 하체 그룹 + 웬들러 열**: 스쿼트 열은 "웬들러 · 기준 150" 배지 + 주 단위 자동 처방(톱세트 kg × n+, BBB 서브라인).
6. **씬 6 — 웬들러 셀 시트**: **한 세션의 순서** — ① 메인 8/6/3 (90→97.5→105×8+, 마지막은 한계까지) ② **바로 이어서 BBB** (기준의 50%=75kg×10×5). 한계 세트 실제 횟수 입력 → 정산 근거. (계약 8)
7. **씬 7 — 6주 정산 시트**: 종목별 `현재 → 다음 (+증량폭)` + [성장|유지](못 채운 종목은 유지 기본). 상체 +2.5/하체 +10 기본값 기반 종목별 설정값 그대로. 웬들러는 TM에 적용.
8. **씬 8 — 종목 설정 시트**: 트랙 구성(볼륨/강도), 세트 수, 6주 성공 시 증량폭(부위 기본값 표시), 운동 방식(기본 계단 ↔ 웬들러: 기준 무게/스킴/주차표/BBB), 여유 횟수(RIR), 자세 메모, 헬스장별 기구.
9. **씬 9 — 종목 추가/삭제**: 그룹 칩 끝 ✚ → 종목 관리 시트. 현재 종목(빼기=보관, 기록 보존) + 운동 라이브러리에서 추가. (계약 12)

## 검증 기준

- **첫 진입 시 빈 보드가 아니다**: 온보딩에 디폴트 종목(테스트모드 기록 기반)이 자동 구성되고, 시작 무게는 기록 있는 종목엔 "X/X 기록 기준"으로 상속 표기, 없는 종목엔 직접 입력 필드가 뜬다. 증량 기본값이 상체 +2.5/하체 +10으로 시드된다.
- **오늘의 배열**: 오늘 행 칸을 탭하면 순서 배지(①②③)가 붙고 하단 배열 바에 쌓인다. 다른 날엔 다른 구성이 가능하다. 안 고른 종목의 그 주 칸은 쉼이 되고 칸 계획은 유지된다.
- 보드에서 과거 6주 밴드가 진녹 색칠 상태로, 현재 밴드가 계획 상태로 보이고 오늘 행이 강조된다.
- 이번 주 셀 탭 → [성공 — 칸 색칠하기] → 셀이 즉시 진녹 + weekLog 저장 + 재로드 후 유지.
- "목표를 못 채웠어요" → "한 주 더 도전" 적용 → 이후 칸들의 주차가 밀려 재계산되고 보드에 반영.
- **웬들러 세션 순서**: 스쿼트 셀 시트가 ① 메인 8/6/3(%TM 3세트, 마지막 한계까지) ② 바로 이어서 BBB(50%×10×5) 순서로 표시되고, 한계 세트 실제 횟수가 기록된다. 정산 시 TM+증량폭(기본 +10).
- 사이클 마지막 주 경과 → 정산 시트 → 종목별 성장/유지 → 다음 6주 칸 생성 + history 1건 추가.
- **종목 추가/삭제**: ✚로 라이브러리에서 추가 → 열이 생기고 다음 주부터 칸 계획 가능. 빼기 → 열이 사라지되 기록 보존, 재추가 시 이어짐.
- **용어 검사**: 화면 어디에도 "중/고", "M/H", "열 메타", "스텝", "W1", "AMRAP" 같은 내부 용어가 노출되지 않는다.
- 줌아웃에서 6개월 범위의 색칠/계획/못 채움이 한 화면에 보인다.
- `_settings.max_cycle`은 어떤 v2 경로에서도 변경되지 않는다 (data-guardian).
