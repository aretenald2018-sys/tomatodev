# Max 벤치마크 그래프: 주차 내 다중 수행점 표시

## 문제

- 현재 계획 조정/성장판의 계단 그래프는 실제 수행선과 점을 표시하지만, 주차별 대표 수행값 1개만 그래프에 연결한다.
- 사용자가 같은 벤치마크 종목을 한 주에 2번 이상 수행하면 두 번째 수행 경과가 그래프에서 사라져, 볼륨/강도 트랙별 진행 추적이 부족하다.

## 목표

- 볼륨 트랙과 강도 트랙 각각에서 해당 트랙으로 기록된 모든 실제 수행점을 그래프에 표시한다.
- 같은 주에 여러 번 수행해도 날짜 순서대로 점을 모두 찍고, 수행선도 그 점들을 이어 경과를 보여준다.
- 주차 상태 라벨은 기존처럼 대표값 1개를 유지해 UI 밀도를 크게 늘리지 않는다.

## 실행 슬라이스

### Slice 1: 렌더 geometry 확장

- `_planTrackPoints()`는 기존 주차 상태와 함께 해당 주의 `actuals` 배열을 보존한다.
- `_stairGeometry()`는 주차 대표 actual이 아니라 모든 actual을 날짜 기준 x/y 좌표로 변환한다.
- 볼륨/강도 트랙 필터링은 기존 `buildBenchmarkActuals({ track })` 결과를 그대로 사용한다.

### Slice 2: 회귀 테스트

- 같은 주에 볼륨 기록 2개가 있으면 `wt-v4-plan-actual-point`가 2개 렌더되는지 확인한다.
- 강도 트랙도 별도 기록 2개가 있으면 강도 lane에서 점이 2개 렌더되는지 확인한다.

### Slice 3: 버전/검증/리뷰

- 변경 정적 자산 쿼리와 service worker cache version을 갱신한다.
- `node --check`, `node --test tests/calc.max.test.js`, `git diff --check`를 통과시킨다.
