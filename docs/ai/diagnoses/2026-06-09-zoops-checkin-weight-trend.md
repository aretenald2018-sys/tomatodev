# 줍스 체중/체지방 추이 진단

## 증상

- `줍스` 계정이 통계 탭의 `체중 & 체지방 추이`에서 체중 표시가 예전과 다르게 이상하다고 보고했다.

## 확인

- 계정 매핑: `줍스` = `최_준수`
- Firestore 원본 경로: `users/최_준수/body_checkins`
- 원본 체중 범위: `2026-04-15` `84.2kg`에서 `2026-06-09` `80.1kg`
- 비정상 체중 범위 값은 없었다.
- 같은 날짜 중복이 있었다.
  - `2026-05-19`: 2건 (`81.2kg`, `81.1kg`)
  - `2026-06-09`: 13건 (`80.1kg` 반복)

## 원인

- 기존 통계 차트는 `getBodyCheckins()` 결과를 그대로 그려 같은 날짜 중복 체크인을 모두 차트 포인트로 사용했다.
- 체크인 저장 버튼을 빠르게 여러 번 누르거나 저장이 느린 상태에서 재요청되면 같은 날짜에 새 `ci_<timestamp>` 문서가 반복 생성될 수 있었다.

## 수정

- 사용자 화면용 `getBodyCheckins()`는 같은 날짜 기록을 최신 `ci_<timestamp>` 1건으로 접어서 반환하게 했다.
- 원본 전체가 필요한 경우를 위해 `getRawBodyCheckins()`를 추가했다.
- 중복 접기 로직은 `data/body-checkins.js` 순수 모듈로 분리해 테스트 가능하게 했다.
- 새 체크인 저장 시 같은 날짜의 기존 최신 기록이 있으면 새 문서를 만들지 않고 그 문서를 업데이트한다.
- 저장 중에는 체크인 저장 버튼을 비활성화해 중복 클릭 저장을 막는다.
- `sw.js` 캐시 버전을 `tomatofarm-v20260609z3-checkin-dedupe`로 올렸다.
