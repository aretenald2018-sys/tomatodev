# 운동 캘린더 바텀시트 클릭 회귀 진단

## 상태

- 날짜: 2026-06-25
- 요청: 바텀시트 화살표/슬라이드에서 기존 클릭 동작까지 깨진 회귀 수정
- 범위: 캘린더 운동 바텀시트 핸들 입력 처리

## 근본 원인

직전 드래그 정착 수정에서 `pointerup.clientY`를 안착 판정에 반영하면서, 동시에 핸들 `pointerdown`에서 `event.preventDefault()`를 호출했다.

모바일 브라우저/PWA에서는 `pointerdown` 기본 동작을 막으면 후속 `click` 합성이 취소될 수 있다. 이 때문에 짧게 클릭해서 위/아래로 토글되던 기존 동작이 회귀했다.

## 수정 방향

`pointerdown`에서는 클릭 생성을 막지 않는다.

실제 드래그 중인 `pointermove`에서만 `preventDefault()`를 호출해 페이지 스크롤 간섭을 막고, `pointerup.clientY` 기반 최종 좌표 판정은 유지한다.

## 완료 기준

- 짧게 클릭하면 기존처럼 바텀시트가 위/아래로 토글된다.
- 드래그/스윽 밀기 후에는 놓은 최종 좌표 기준으로 full/bar에 안착한다.
- 이동 중에는 문서 스크롤이 끼어들지 않는다.
